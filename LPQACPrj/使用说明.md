# QAC 静态代码扫描报告处理工具 - 使用说明

## 概述

本工具用于处理 QAC 静态代码扫描生成的 Diagnostic Summary HTML 报告，可以按需删除指定 Severity 等级的项，同时保持 HTML 格式完整。

## 功能特点

- ✅ 自动扫描目录中所有 HTML 报告文件
- ✅ 交互式选择要删除的 Severity 值
- ✅ 保持 HTML 格式完整，不影响其他内容
- ✅ 删除 Severity 时，会同时删除其下所有子项（Message 和 File）
- ✅ 支持批量处理多个文件
- ✅ 操作前确认机制，防止误删

## 文件说明

- `process_all_html.py` - 主处理脚本

## 系统要求

- Python 3.x
- 无需额外依赖库（仅使用 Python 标准库）

## 使用步骤

### 1. 准备文件

确保要处理的 HTML 文件与脚本在同一目录下。

### 2. 运行脚本

在命令行中执行：

```bash
python process_all_html.py
```

### 3. 查看可用 Severity 值

脚本会自动扫描所有 HTML 文件，并显示找到的所有 Severity 值，例如：

```
找到 4 个HTML文件:
  1. LPAC014-APP-静态扫描报告-ASW.html
  2. LPAC014-APP-静态扫描报告-CDD.html
  3. LPAC015-APP-静态扫描报告-ASW.html
  4. LPAC015-APP-静态扫描报告-CDD.html

正在扫描所有文件中的Severity值...

找到以下Severity值: [1, 2, 3, 4, 5, 6, 7, 8]
```

### 4. 选择要删除的 Severity 值

根据提示输入要删除的 Severity 值：

```
请选择要删除的Severity值（输入数字，多个用逗号或空格分隔，例如: 6,7,8 或 6 7 8）
输入 'all' 删除所有，输入 'none' 或直接回车取消操作
请输入: 
```

### 5. 确认操作

输入要删除的 Severity 值后，脚本会显示确认信息：

```
将删除以下Severity值: [6, 7, 8]

确认删除? (y/n): 
```

输入 `y`、`yes`、`是` 或 `确认` 来确认删除，输入其他内容则取消操作。

### 6. 查看处理结果

脚本会显示每个文件的处理结果：

```
处理: LPAC014-APP-静态扫描报告-ASW.html
  原始大小: 1,556,297 字符
  删除: 300,302 字符
  新大小: 1,255,995 字符
  已保存: LPAC014-APP-静态扫描报告-ASW.html
```

```

```

或直接按回车键。

## 注意事项

### ⚠️ 重要提示

1. **文件会被覆盖**：脚本会直接修改原 HTML 文件，建议在处理前备份重要文件。

2. **不可逆操作**：删除操作无法撤销，请谨慎选择要删除的 Severity 值。

3. **保留的子项**：删除某个 Severity 后，其他 Severity 下的所有子项（Message 和 File）都会完整保留。

4. **HTML 格式**：脚本会保持 HTML 格式完整，不会破坏文件结构。

### 使用建议

- 首次使用前，建议先用测试文件验证效果
- 处理重要文件前，建议先备份
- 确认要删除的 Severity 值后再执行操作

## 常见问题

### Q1: 脚本提示"未找到HTML文件"

**A:** 确保 HTML 文件与脚本在同一目录下，且文件扩展名为 `.html`。

### Q2: 输入了不存在的 Severity 值会怎样？

**A:** 脚本会显示警告信息并忽略该值，例如：
```
警告: Severity 10 不在可用列表中，已忽略
```

### Q3: 可以恢复已删除的内容吗？

**A:** 不可以。删除操作会直接修改原文件，无法恢复。建议处理前备份文件。

### Q4: 脚本会处理哪些文件？

**A:** 脚本会自动处理当前目录下所有 `.html` 文件。

### Q5: 删除 Severity 后，其他 Severity 的内容会受影响吗？

**A:** 不会。脚本只删除指定的 Severity 及其子项，其他 Severity 的内容会完整保留。

## 技术说明

### 工作原理

1. 脚本使用正则表达式查找所有包含 `severity_X` 的 HTML 元素
2. 对于每个要删除的 Severity，找到包含它的完整 `<tr>` 块
3. 处理嵌套的表格结构，确保删除完整的 Severity 块
4. 合并重叠的块，避免重复删除
5. 从后向前删除，避免位置偏移问题

### 支持的 HTML 结构

脚本能够正确处理以下 HTML 结构：
- Severity 主行（包含 Severity 链接的行）
- Child group 行（包含子项的行）
- 嵌套的表格结构
- 多个 Message 和 File 子项
